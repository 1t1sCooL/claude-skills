# Мои кастомные скиллы — для чего всё это

13 персональных скиллов под рабочий процесс фронтенд-фрилансера: найти клиента →
закрыть → сделать качественно → задеплоить → вырасти. Ниже — что каждый делает,
когда звать и что на выходе. Скиллы соединяются в цепочки (см. «Как связано»).

---

## 🤝 Закрытие клиентов (низ воронки после `frontend-audit`)

| Скилл | Когда | Выход |
|---|---|---|
| **`/proposal`** | Аудит готов, надо выслать КП. Вход: результат `frontend-audit` / URL / описание. | `./proposals/<client>.md`: проблемы→влияние, 3 тарифа, цены, сроки, CTA. |
| **`/outreach-track`** | Ведёшь холодные касания, чтобы никто не потерялся. `add`/`touch`/`due`/сводка. | `./outreach/outreach.md` + `.csv`, шаблоны 3 касаний, кого пинговать сегодня. |
| **`/invoice`** | Сделка закрыта — выставить счёт. Вход: proposal или позиции. | `./billing/invoices/<num>.md` + печатный `.html`, автонумерация (РФ, без НДС по умолч.). |

## 🔬 Качество фронта (гейт перед сдачей/деплоем)

| Скилл | Когда | Выход |
|---|---|---|
| **`/perf-guard`** | Реально замерить скорость и поймать регрессию. **Реальный Lighthouse/PSI vs прод**, не chrome-devtools trace (он врёт). | `./perf-guard-output/report.md` + JSON, pass/fail по бюджету, дифф «было→стало». |
| **`/a11y`** | Проверить доступность по WCAG (axe-core + ручной чеклист). | `./a11y-output/report.md`, находки по severity с WCAG SC и фиксами. |
| **`/seo-tech`** | Техническое SEO страницы (мета, OG, JSON-LD, canonical, robots, sitemap, hreflang). | `./seo-tech-output/report.md`, чеклист ✓/✗ с фиксами. |

## 🛠 Dev-продуктивность (ежедневный цикл)

| Скилл | Когда | Выход |
|---|---|---|
| **`/slice <layer> <name>`** | Завести новый FSD-слайс по конвенциям текущего проекта (детектит TS/CSS Modules/barrel сам). | Папка слоя + сегменты + компонент + `index.ts`. |
| **`/deploy [staging\|prod]`** | Релиз под Jenkins+Traefik+K8s: предполёт → триггер → пост-чек **сжатия** (double-Ingress gotcha) + smoke. | Отчёт прогона + чеклист пост-деплоя. |
| **`/changelog`** | Собрать релиз-ноты из git (Conventional Commits → секции). `--write` пишет в `CHANGELOG.md`. | Markdown-секция релиза. |

## Как связано (цепочки)

- **Воронка денег:** `/frontend-audit` (найти лида) → `/proposal` (КП) → `/outreach-track` (дожать касаниями) → `/invoice` (счёт).
- **Гейт качества перед сдачей:** `/perf-guard` + `/a11y` + `/seo-tech` → находки в `/proposal` как обоснование цены, или как чеклист перед `/deploy`.
- **Dev-цикл:** `/slice` (новый код) → … → `/deploy` (релиз) → `/changelog` (ноты).
- **Личный бренд:** рабочая сессия → `devlog` (агент) → `/devlog-post` (публикация) + `/flashcards` (запоминание).
- **Рост:** `/growth-audit` (где теряю) → `/seo-content` + `/seo-tech` (органика сайта) + `/tg-content` (канал).

## Настройка (один раз, что попросят при первом запуске)

- **`invoice`** — заполнить `./billing/profile.json` (реквизиты). Секреты — в `~/.claude/secrets.env`.
- **`tg-content` / `growth-audit`** — указать хэндл ТГ-канала → сохранится в `./growth/config.json`.
- **`perf-guard`** — нужен Chrome + Lighthouse (CLI) или ключ PSI API (`PSI_API_KEY`).
- **`a11y`** — Playwright для axe-core; запускать **из главной сессии** (сабагентам браузер закрыт).
- **`deploy`** — настроить плейсхолдеры топологии (Jenkins job, домены, smoke-роуты) под проект.
