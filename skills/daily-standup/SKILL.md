# Daily Standup Report

Пишет дневной отчёт о проделанной работе в стиле пользователя — для менеджера, без техножаргона.

## Что выдать

Только сам отчёт, без вступлений и пояснений. Формат:

```
<день>.<месяц>
- <пункт>
- <пункт>
```

- **Заголовок-дата** на отдельной строке: `день.месяц` без ведущих нулей и без года (например `16.6`). По умолчанию — сегодня; если пользователь назвал дату/«вчера» — берём её.
- Каждый пункт — `- ` + одна короткая строка.
- **Один пункт = одна логическая задача**, а не файл/коммит. Несколько коммитов одной темы сворачиваем в один пункт.

## Стиль (строго соблюдать)

- Русский, **первое лицо, прошедшее время**: «Оптимизировал», «Заменил», «Добавил», «Решил», «Сделал», «Актуализировал».
- **Бизнес-понятный язык, без технического жаргона.** Описываем эффект/смысл, а не реализацию.
  - ❌ «Вынес JSON виджетов в RSC-обёртку, next/image AVIF srcset, preconnect»
  - ✅ «Оптимизировал главную: страница грузит меньше лишнего кода»
  - ✅ «Переход на более лёгкие картинки»
- Незавершённое помечаем `(in progress)`.
- Уточняющий контекст — в скобках: `(qiaopay)`, `(онлайн сервисы)`.
- Кратко: без точек в конце, без подпунктов, без эмодзи.

## Как собрать, что было сделано

1. Определи день и автора:

```bash
git log --author="$(git config user.name)" --since="<YYYY-MM-DD> 00:00" --until="<YYYY-MM-DD> 23:59" --pretty="- %s" --no-merges
```

(для «сегодня»/«вчера» подставь нужную дату; мердж-коммиты исключаем).

2. Дополни контекстом текущей сессии чата (что реально делали руками, даже если ещё не закоммичено).
3. Переведи каждую тему в один бизнес-понятный пункт по правилам стиля выше.
4. Если задача очевидно не закончена — пометь `(in progress)`.

## Эталон стиля (few-shot)

Так пишет пользователь — копируй интонацию и уровень детализации:

```
15.6
- Оптимизировал несколько остальных страниц лендинга (переход на более легкие картинки)
- Заменил все кавычки на елочные (французские)
- Актуализация содержания страниц (in progress)
- Решил проблему с /meta.json
- Добавил базовую валидацию на форму обратной связи
- Сделал карты одинакового размера (qiaopay)
- Добавил якори на секцию онлайн сервисов
```

```
16.6
- Оптимизировал главную и связанные страницы: вынес контент виджетов из клиентского бандла (стало меньше JS на загрузку и парсинг)
- Добавил preconnect/dns-prefetch для аналитики и виджета отзывов (ускоряет подключение)
- Подчистил лишние данные в слайдерах схемы и оплаты (на страницу уходит меньше веса)
```
