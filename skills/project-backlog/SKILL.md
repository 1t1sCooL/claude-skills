# Project Backlog

Генерирует список задач, которые стоит решить в проекте — для быстрого планирования.

## Что выдать

Только список, без вступлений и пояснений. Формат:

```
<день>.<месяц>
- <задача>
- <задача>
```

- **Заголовок-дата** — сегодня, `день.месяц` без ведущих нулей (например `16.6`).
- Каждый пункт — `- ` + одна строка. Без точек в конце, без подпунктов, без эмодзи.
- **5–10 задач**, от важных к менее важным.

## Стиль

- Русский, **назывные предложения или короткий глагол**: «Оптимизация остальных страниц», «Регрессионные тесты», «Добавить A/B тест».
- Бизнес-понятный язык — эффект/смысл, не реализация.
  - ❌ «Мигрировать raw img в next/image для premium-виджетов»
  - ✅ «Оптимизация страницы /plastic (картинки)»
- Уточняющий контекст — в скобках: `(блог)`, `(главная)`, `(qiaopay)`.
- Незапущенное важное — в начале списка.

## Как собрать задачи

Используй **все** доступные источники — чем больше контекста, тем точнее список:

### 1. ROADMAP.md
```bash
cat .ai-factory/ROADMAP.md 2>/dev/null | head -100
```
Ищи незакрытые milestone и deferred-пункты (M15+, `[ ]`, "отложено", "TODO").

### 2. Бэклог в планах
```bash
cat .ai-factory/PLAN.md 2>/dev/null | grep -A 50 "Backlog\|backlog\|следующ\|кандидат"
ls .ai-factory/plans/ 2>/dev/null
```
Секции `## Backlog` и `## Следующие кандидаты` в PLAN.md — прямой источник.

### 3. Git-история (что уже сделано → понять что осталось)
```bash
git log --oneline -20 --no-merges
```
Если видишь серию оптимизаций/фиксов в одной зоне — скорее всего там есть продолжение.

### 4. Открытые TODO/FIXME в коде
```bash
grep -r "TODO\|FIXME\|HACK\|in progress" src/ --include="*.ts" --include="*.tsx" -l 2>/dev/null | head -10
```

### 5. Контекст текущей сессии
Что обсуждали / делали в этом чате, но ещё не завершили или отложили — добавь как задачи.

### 6. memory (если есть)
Проверь memory проекта на незакрытые инициативы: `.claude/projects/*/memory/MEMORY.md`.

## Синтез

После сбора данных:
1. Сгруппируй похожие темы в один пункт
2. Убери то, что уже закрыто (есть в git как выполненное)
3. Расставь по приоритету: сначала то, что влияет на пользователей / метрики
4. Сформулируй каждый пункт коротко и по-человечески

