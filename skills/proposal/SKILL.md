# Commercial Proposal (КП)

Генератор коммерческого предложения для услуг фронтенд-разработки/аудита. Замыкает воронку
`frontend-audit` → деньги: берёт найденные проблемы и превращает их в КП с обоснованием цены,
тарифами и понятным следующим шагом. Тон — конструктивный, через ценность для бизнеса, без давления.

Услуги автора: фронтенд-разработка, аудит и ускорение сайтов (`mmalabugin.ru`, страница аудита `mmalabugin.ru/audit`).

## Что на входе (`$ARGUMENTS`)

Один из:
- **Путь к результату `frontend-audit`** (`.md` / `.json`) → взять проблемы оттуда.
- **URL клиента** → кратко прозондировать через WebFetch (что за бизнес, очевидные слабые места) — не полноценный аудит, а контекст.
- **Свободное описание scope** → построить КП из него.
- **Пусто** → спросить через AskUserQuestion: что на входе (аудит / URL / описание).

## Шаги

1. **Собрать проблемы и контекст.** Из аудита/зонда/описания вытащить: что за клиент, ключевые проблемы (perf, SEO, конверсия, доступность, тех-долг), и **во что это обходится бизнесу** (отказы из-за скорости, потеря трафика, низкая конверсия CTA).

2. **Сформировать блок «проблема → влияние».** По каждой находке: коротко проблема + следствие в деньгах/метриках (без запугивания, по делу).

3. **Описать scope работ** — что конкретно делаем (список работ под найденные проблемы).

4. **Три тарифа** с конкретными deliverables:
   - **Базовый** — закрыть самое критичное (быстрая победа).
   - **Стандарт** — комплекс: критичное + значимое (рекомендуемый).
   - **Премиум** — всё + сопровождение/повторный аудит «было→стало».
   Цены — **вилки-плейсхолдеры** (`<от–до ₽>`) с пометкой «настроить под свой прайс»; не выдумывать конкретные суммы.

5. **Сроки** по каждому тарифу (диапазон) + **next-step CTA**: короткий созвон / заполнить бриф / ссылка `mmalabugin.ru`. Реквизиты/контакты — плейсхолдеры с пометкой заполнить.

## Выход

- `./proposals/<client-slug>.md` — готовое КП: проблемы→влияние, scope, 3 тарифа с deliverables и вилками цен, сроки, CTA.
- Короткая сводка: что подсветить на созвоне, какой тариф предлагать как основной.
- Список плейсхолдеров для заполнения (цены, контакты, реквизиты).
