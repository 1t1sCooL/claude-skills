# SEO Content (mmalabugin.ru)

**Контентная** половина SEO для сайта автора (фронтенд-разработка, аудит и ускорение сайтов).
Техническая часть — отдельный скилл `seo-tech`. Только белые методы: семантика → интенты →
контент-план → мета под выдачу. Никакой накрутки/клоакинга.

## Что на входе (`$ARGUMENTS`)

- Опц. **тема/направление услуг** (`аудит сайта`, `ускорение`, `Core Web Vitals`, `фронтенд на заказ`). Пусто → по основным услугам.

## Шаги

1. **Семантика.** Через WebSearch собрать реальные запросы/интенты по нише: «аудит сайта», «ускорить сайт», «оптимизация Core Web Vitals», «заказать лендинг», «почему сайт медленный» и т.п. — вместе с близкими формулировками.
2. **Кластеры по интенту:**
   - **Информационные** — «почему сайт тормозит», «что такое LCP» → статьи/блог (верх воронки).
   - **Коммерческие** — «заказать аудит сайта», «ускорение сайта цена» → услуговые страницы (близко к деньгам).
   - **Брендовые** — имя/проекты автора.
3. **Контент-план** — какие страницы/статьи закрывают какие кластеры; **приоритет = спрос × близость к деньгам** (сначала коммерческие с реальным спросом).
4. **Мета под выдачу** для ключевых страниц: `title` (≤60), `description` (≤155, с УТП и мягким CTA), `H1`, структура `H2`; рекомендации по **внутренней перелинковке** (инфо-статьи → услуговые страницы).

## Выход

- `./growth/seo-content-<date>.md` — кластеры по интентам, приоритизированный контент-план, мета-тексты ключевых страниц, схема перелинковки.
- Что отдать в `seo-tech` (техзадачи) и какие 2–3 страницы делать первыми.
